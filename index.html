Case Scenario

"This was not a classroom exercise — I was assigned a real evidence file and asked to build a case around it, just as a working forensic analyst would."

The scenario: employed as a Digital Forensics Analyst at a cybersecurity company, I was assigned by my line manager to investigate a piece of digital evidence recovered from a suspect's storage device. The brief was clear — treat this as a real investigation. Maintain chain-of-custody at every step. Produce a report that could stand up in court.

The digital evidence was provided via the F21FO course Canvas portal. My task: create forensically sound images using two separate imaging tools, then perform deep analysis using two separate analysis tools, documenting every step with timestamps, actions, and findings.

The final deliverable — a professional technical report and a 5-minute demonstration video — was assessed against the same criteria used in real forensic analysis: thoroughness, evidence integrity, technical accuracy, and quality of reflection.

Why Dual Tools? NIST guidelines recommend using multiple tools to cross-validate findings — because no single forensic tool is infallible. Discrepancies between tools can indicate either a tool limitation or a deeper evidence anomaly worth investigating.
Chain of Custody: Every action taken on the evidence was timestamped and documented — from initial receipt, through imaging, to analysis. This is what separates a forensic investigation from a simple file scan: the evidence can be traced, verified, and defended.
Distinction Grade: This coursework was awarded a Distinction (70%+) — the highest grade band at Heriot-Watt — for thoroughness of analysis, chain-of-custody documentation, and quality of the technical report and demonstration.

Tools Used

Primary Imaging Tool
FTK Imager
Forensic Imaging · Exterro · Industry Standard

Used to create the primary forensically sound disk image. FTK Imager creates bit-for-bit copies of storage media with cryptographic hash verification (MD5/SHA-1) — ensuring the forensic image is provably identical to the original evidence. Used by law enforcement and forensic labs globally.

Used for: Primary disk imaging · Hash generation · Evidence preservation
Secondary Imaging Tool
OSForensics
Forensic Suite · PassMark Software · Windows

Used as the second imaging method to cross-validate the FTK image. OSForensics provides high-performance file search, indexing, and imaging — including hash matching, registry analysis, MFT analysis, timeline analysis, and memory analysis. The dual-image approach confirms evidence integrity.

Used for: Secondary imaging · Hash verification · Cross-validation
Primary Analysis Tool
Autopsy
Forensic Analysis · Open Source · GUI Platform

The primary analysis platform — providing file carving, keyword search, timeline analysis, hash lookup, recent activity, email parsing, and encryption detection. Autopsy's GUI enables systematic, documented examination of the forensic image with structured reporting output.

Used for: File analysis · Timeline · Hash lookup · Report generation
Secondary Analysis Tool
Bulk Extractor
Data Extraction · Command Line · Cross-Platform

A forensic tool that scans disk images linearly — bypassing the file system entirely to extract emails, URLs, credit card numbers, GPS coordinates, and network data directly from raw bytes. Particularly powerful for detecting data that file-system-based tools miss.

Used for: Email extraction · Network analysis · Metadata · Social media data

Investigation Methodology

01
Evidence Receipt & Chain-of-Custody Initiation
Digital evidence received from the F21FO Canvas portal. Chain-of-custody log initiated immediately — recording receipt timestamp, evidence identifier, storage location, and investigator name. No modifications made to evidence before imaging. Evidence hash recorded for later verification.
NIST Principle: Preserve evidence integrity before any analysis begins
02
Forensic Imaging — FTK Imager
Created a bit-for-bit forensic image of the evidence using FTK Imager. Generated MD5 and SHA-1 hashes of both the original evidence and the resulting image — confirming identical bit-level copies. Image saved in .E01 (Expert Witness Format) for maximum metadata preservation. Chain-of-custody log updated with imaging start/end timestamps.
Tool: FTK Imager · Format: .E01 · Verification: MD5 + SHA-1 hash match
03
Forensic Imaging — OSForensics (Cross-Validation)
Created a second forensic image using OSForensics to independently verify imaging integrity. Compared hash values between FTK and OSForensics images — confirming evidence was not modified during the imaging process. Discrepancies between imaging results would have flagged potential evidence tampering or tool error.
Tool: OSForensics · Purpose: Cross-validation · Result: Hash values matched
04
Deep Analysis — Autopsy
Loaded forensic image into Autopsy for systematic examination. Ran file analysis, keyword search, timeline analysis, hash lookup (against known malicious hash databases), recent activity analysis, email parsing, and encryption detection. Identified files of interest and documented findings with file paths, timestamps, and metadata.
Techniques: File carving · Timeline · Hash lookup · Email parser · Encryption detection
05
Data Extraction — Bulk Extractor
Applied Bulk Extractor to scan the forensic image at the raw byte level — extracting email addresses, URLs, network traffic data, GPS coordinates, and social media artefacts. This file-system-agnostic approach recovered data that Autopsy's file-system-based analysis could not access — demonstrating the complementary value of dual-tool analysis.
Techniques: Linear scan · Email extraction · Network analysis · Histogram analysis
06
Technical Report & Video Demonstration
Produced a comprehensive technical report documenting the full investigation — introduction, forensic imaging process, analysis findings, chain-of-custody appendix, and critical reflection. Submitted alongside a 5-minute demonstration video showcasing imaging and analysis in action. Report structured to meet court-submission standards.
Deliverables: Technical report (8 pages) + 5-minute screencast demonstration

Analysis Techniques Demonstrated

File Recovery
Scenario: A McDonald's employee accidentally formatted a USB device containing confidential product launch data. Using Autopsy's PhotoRec carving, keyword search, and file analysis features, all deleted files were recovered from unallocated space — demonstrating that formatting does not destroy data.
Tool: Autopsy · Technique: File carving + unallocated space scan
Hash Analysis
Scenario: PwC suspected an employee of tampering with or stealing data via USB. Using Autopsy's hash lookup feature with a pre-existing known-file hash database, tampered files were automatically flagged — distinguishing original files from modified copies without manual comparison.
Tool: Autopsy · Technique: MD5/SHA-1 hash comparison · Known-bad database
Entropy Analysis
Scenario: Google's security team found suspicious encrypted files on an employee's machine. Using Autopsy's encryption detection and file metadata analysis, files protected with 7-Zip password encryption were identified and flagged — providing investigators with the evidence needed to establish intent.
Tool: Autopsy · Technique: Encryption detection + metadata analysis
Email Analysis
Scenario: McDonald's security team suspected data exfiltration via email. Using Bulk Extractor's email analysis feature, all outbound emails from the employee's machine were extracted — including headers, content, and recipient addresses. Evidence of corporate data being sent to a competitor was recovered.
Tool: Bulk Extractor · Technique: Email header + content extraction
Network Analysis
Scenario: Unauthorised data was transferred from Heriot-Watt's Robotarium lab network. Network traffic was captured using tcpdump and analysed using Bulk Extractor's network analysis module — extracting IP histograms and identifying a suspicious external IP address, which was then blocked.
Tool: Bulk Extractor · Technique: PCAP analysis + IP histogram
Timeline Analysis
Using Autopsy's timeline analysis module, all file system events (creation, modification, access, deletion) were mapped chronologically — enabling reconstruction of the suspect's activity sequence. Timeline evidence is critical for establishing intent and motive in court proceedings.
Tool: Autopsy · Technique: File system event timeline reconstruction
